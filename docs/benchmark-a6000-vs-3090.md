# Benchmark: NVIDIA RTX A6000 vs RTX 3090

## 환경

- 모델: Qwen3-TTS-12Hz-0.6B-Base, Qwen3-TTS-12Hz-1.7B-Base
- 추론 방식: CUDA Graph fast path vs Dynamic cache baseline
- 측정 항목: TTFA (Time to First Audio), RTF (Real-Time Factor)

## TTFA — chunk_size=8, fast path (ms, 낮을수록 좋음)

| 모델 | RTX A6000 | RTX 3090 | 차이 |
|------|-----------|----------|------|
| 0.6B | 279ms | **229ms** | 3090이 18% 빠름 |
| 1.7B | 303ms | **250ms** | 3090이 18% 빠름 |

## TTFA — chunk_size별 fast path (ms)

| chunk_size | A6000 0.6B | 3090 0.6B | A6000 1.7B | 3090 1.7B |
|-----------|-----------|----------|-----------|----------|
| 4 | 200ms | **153ms** | 212ms | **164ms** |
| 8 | 279ms | **229ms** | 303ms | **250ms** |
| 12 | 364ms | **305ms** | 400ms | **337ms** |

chunk_size=4가 최저 지연이지만 오디오 품질 트레이드오프가 있을 수 있음.

## RTF — fast path (높을수록 좋음, 1.0 = 실시간)

| 모델 | RTX A6000 | RTX 3090 | 차이 |
|------|-----------|----------|------|
| 0.6B | 3.06x | **3.61x** | 3090이 18% 높음 |
| 1.7B | 2.71x | **3.25x** | 3090이 20% 높음 |

## RTF — baseline (dynamic cache)

| 모델 | RTX A6000 | RTX 3090 |
|------|-----------|----------|
| 0.6B | 0.43x | **0.52x** |
| 1.7B | 0.42x | **0.51x** |

baseline은 RTF < 1.0 이므로 실시간 미만. fast path 필수.

## 실서버 스트리밍 TTFA (네트워크 포함)

모델: 1.7B, 텍스트: 한국어 57자 (경동나비엔 콜봇 인사말)

| GPU | 서버 주소 | 네트워크 포함 TTFA |
|-----|-----------|------------------|
| RTX 3090 | 172.31.79.202:8000 | **584.68ms** |
| RTX A6000 | 172.31.88.110:8001 | 608.49ms |

- 로컬 벤치마크(네트워크 제외) 대비 약 300ms 추가 지연 발생 (네트워크 RTT + HTTP 오버헤드)
- 실환경에서도 3090이 소폭 빠름 (~24ms 차이)

## 분석

**RTX 3090이 전 항목에서 우세**하다. A6000은 VRAM(48GB)이 더 크지만, 메모리 대역폭에서 3090에 뒤진다.

| 항목 | RTX A6000 | RTX 3090 |
|------|-----------|----------|
| VRAM | 48 GB | 24 GB |
| 메모리 대역폭 | 768 GB/s | 936 GB/s |

TTS 추론은 대규모 행렬 연산보다 메모리 대역폭 바운드 워크로드에 가깝기 때문에, 대역폭이 높은 3090이 유리하다. A6000의 대용량 VRAM은 더 큰 배치나 더 긴 시퀀스 처리에 활용할 수 있다.

## 결론

- **지연 시간 우선 (실시간 스트리밍)**: RTX 3090 권장, chunk_size=4로 TTFA ~153ms 달성 가능
- **대용량 처리 / 배치 추론**: A6000의 48GB VRAM 활용 가능
